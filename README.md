# Система защищенного удаленного доступа к корпоративной сети

Это учебный проект — мобильное VPN-приложение, которое я делаю в рамках учебы. Идея в том, что есть мобильный клиент (Flutter) и бэкенд-сервер (Python + FastAPI), который раздает конфигурации подключения. Клиент сам подключается к VPN через ядро V2Ray.

Проект представляет собой клиент-серверное приложение для организации безопасного сетевого туннеля. Основная цель — обеспечить удаленный доступ к внутренним ресурсам сервера, используя современные протоколы шифрования.

## Технологический стек

### Клиент 
*   **Фреймворк:** Flutter (Dart) — для кроссплатформенной разработки
*   **Интерфейс:** Material Design 3 — тёмная тема, выглядит как нормальный VPN-клиент
*   **Сетевое ядро:** flutter_v2ray (плагин поверх V2Ray/Xray) — он поднимает реальный VPN-туннель на Android
*   **Взаимодействие с API:** пакет `http` — стучится на наш FastAPI и тянет список серверов в формате JSON

### Серверная часть 
*   **Управляющий сервер:** Python (FastAPI) — запускается командой `uvicorn`, отдает JSON с серверами по маршруту `/servers`
*   **Виртуальное окружение:** `.venv` — все зависимости Python изолированы, чтобы не засорять систему
*   **Сервер доступа:** Развернут на удаленном сервере под управлением Linux. Использует ядро Xray/Sing-box для терминации трафика
*   **Контейнеризация:** Docker — для быстрого развертывания и изоляции сервисов на управляющем сервере и серверах доступа

## Архитектура системы
Система делится на 3 части (я сначала думал что 2, но потом понял что клиент — это отдельный уровень):

### 1. Уровень управления (FastAPI бэкенд)
Центральный узел, расположенный в надежном месте, способный поддерживать бесперебойную работу. По сути это небольшой Python-сервер на FastAPI, который я запускаю через uvicorn.

*   **Задача:** Хранение списка доступных серверов и генерация персональных настроек подключения.
*   **Принцип работы:** Приложение обращается к API (`GET /servers`), получает список серверов в виде JSON. Каждый сервер содержит список конфигов — это строки вида `vless://...` или `ss://...`. Это позволяет администратору менять настройки без обновления приложения у клиентов.
*   **Пример ответа сервера:**
```json
[
  {
    "id": "server-1",
    "name": "Germany 🇩🇪",
    "location": "Frankfurt",
    "flagUrl": "https://flagcdn.com/w80/de.png",
    "ip": "1.2.3.4",
    "ping": 45,
    "configs": [
      {
        "protocol": "vless",
        "config_string": "vless://uuid@1.2.3.4:443?...",
        "priority": 1
      }
    ]
  }
]
```

### 2. Уровень данных (VPN-серверы)
Сервера, через которые проходит пользовательский трафик.

*   **Протоколы:** Поддерживается работа с несколькими современными транспортными протоколами (VLESS, Shadowsocks-2022, Hysteria2 и тд). В модели `VpnServer` есть enum `Protocol` с вариантами: `vless`, `shadowsocks`, `wireguard`, `fptn`.
*   **Автоматический выбор:** Клиентское приложение берёт первый конфиг из списка (по полю `priority`). В будущем планирую сделать тестирование задержки и автопереключение на лучший протокол.

### 3. Мобильное приложение (Flutter)
Клиентская часть, установленная на устройство пользователя. Написана на Dart с использованием Flutter. Работает на Android (iOS пока не тестировал).

*   **Функционал:**
    *   Главный экран (`main.dart`) — большая круглая кнопка включения/выключения VPN.
    *   Отображение статуса: DISCONNECTED / CONNECTING... / CONNECTED — с разными цветами.
    *   Нижняя панель со статистикой: скорость скачивания, загрузки и пинг выбранного сервера.
    *   Экран выбора сервера (`server_selection_screen.dart`) — список серверов с флагами и пингом, тянется с FastAPI.

## Как запустить

### Бэкенд (FastAPI)

1. Активировать виртуальное окружение:
```bash
# Windows
.venv\Scripts\activate

# Linux/Mac
source .venv/bin/activate
```

2. Запустить сервер:
```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

После этого сервер будет доступен по адресу `http://127.0.0.1:8000`. Если запускать на Android-эмуляторе, то в `api_service.dart` нужно поменять адрес на `10.0.2.2` — это особенность эмулятора, так он видит локальный компьютер.

### Мобильное приложение (Flutter)

```bash
cd UI
flutter pub get
flutter run
```

Перед запуском на Android убедись что девайс подключен или эмулятор запущен.

## Особенности
*   **Отказоустойчивость:** Архитектура позволяет горячую замену шлюзов доступа без прерывания обслуживания пользователей.
*   **Гибкость:** Поддержка различных протоколов передачи данных позволяет системе адаптироваться к разным условиям сетевой среды (например, нестабильный Wi-Fi или мобильная сеть).
*   **Централизованное управление:** Все настройки подключения формируются на сервере, что упрощает администрирование парка устройств.
