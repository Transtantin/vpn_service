# Система защищенного удаленного доступа к корпоративной сети

Проект представляет собой клиент-серверное приложение для организации безопасного сетевого туннеля. Основная цель — обеспечить удаленный доступ к внутренним ресурсам сервера, используя современные протоколы шифрования.

## Технологический стек

### Клиент 
*   **Фреймворк:** Flutter (Dart) — для кроссплатформенной разработки
*   **Интерфейс:** Material Design 3 — UI
*   **Сетевое ядро:** Sing-box
*   **Взаимодействие с API:** — для получения актуальных конфигураций

### Серверная часть 
*   **Управляющий сервер:** Python (FastAPI). Отвечает за аутентификацию пользователей и распределение конфигурационных файлов
*   **Сервер доступа:** Развернут на удаленном сервере под управлением Linux. Использует ядро Xray/Sing-box для терминации трафика
*   **Контейнеризация:** Docker — для быстрого развертывания и изоляции сервисов на управляющем сервере и серверах доступа

## Архитектура системы
Система делится на 2 уровня:

### 1. Уровень управления 
Центральный узел, расположенный в надежном месте, способный поддерживать бесперебойную работу.
*   **Задача:** Хранение списка доступных серверов и генерация персональных настроек подключения.
*   **Принцип работы:** Приложение обращается к API, проходит авторизацию и получает актуальный список серверов в формате JSON. Это позволяет администратору менять настройки сети без необходимости обновлять само мобильное приложение у клиентов.

### 2. Уровень данных 
Сервера, через которые проходит пользовательский трафик.
*   **Протоколы:** Поддерживается работа с несколькими современными транспортными протоколами (VLESS, Shadowsocks-2022, Hysteria2 и тд).
*   **Автоматический выбор:** Клиентское приложение реализует тестирование задержки. Оно периодически опрашивает доступные протоколы и автоматически переключается на тот, который обеспечивает наилучшую скорость соединения и стабильность в данный момент времени.

### 3. Мобильное приложение
Клиентская часть, установленная на устройство пользователя.
*   **Функционал:**
    *   Кнопка быстрого подключения/отключения.
    *   Отображение текущего статуса соединения и статистики (скорость, пинг).
    *   Возможность ручного выбора сервера или режима "Авто".


## Особенности
*   **Отказоустойчивость:** Архитектура позволяет горячую замену шлюзов доступа без прерывания обслуживания пользователей.
*   **Гибкость:** Поддержка различных протоколов передачи данных позволяет системе адаптироваться к разным условиям сетевой среды (например, нестабильный Wi-Fi или мобильная сеть).
*   **Централизованное управление:** Все настройки подключения формируются на сервере, что упрощает администрирование парка устройств.
